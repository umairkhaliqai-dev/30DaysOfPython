🧠 The Big Picture First
Think of MongoDB like Excel in the cloud but for code.

Your Python code → writes/reads data → MongoDB stores it → forever, 
even after you close Python

Without a database, every time you close Python, all your data is gone. 
MongoDB solves that.

1️⃣ What IS MongoDB?
Forget technical words. MongoDB is just a place to store your data as 
Python dictionaries.
python# This is what MongoDB stores — just a dict!
{"name": "Alice", "age": 20, "grade": "A"}
That's it. No complicated tables, no rows and columns like Excel. Just dictionaries.

2️⃣ SQL vs NoSQL — One Sentence Each
SQLData in rigid tables — like Excel, every row must have same columnsMongoDBData 
as flexible dicts — each document can have different fields

3️⃣ Atlas = MongoDB in the Cloud
Instead of installing MongoDB on your laptop, Atlas hosts it on the internet for
 free. Your Python connects to it via the URI — think of it as a 
 website address + password combined.
mongodb+srv://username:password@cluster.mongodb.net/

5️⃣ The 4 Things You Do With Any Database
Everything in files 06-14 is just these 4 operations:
python# CREATE — add data
students.insert_one({"name": "Alice", "age": 20})
students.insert_many([{...}, {...}, {...}])

# READ — get data
students.find()                        # get all
students.find({"grade": "A"})          # get filtered
students.find().sort("age", 1).limit(3) # sorted + limited

# UPDATE — change data
students.update_one({"name": "Alice"}, {"$set": {"grade": "A+"}})

# DELETE — remove data
students.delete_one({"name": "Alice"})
students.drop()  # delete entire collection
That's literally everything files 06-14 taught. Just these 4 operations with different variations.

6️⃣ Query Operators — Just Comparisons
python{"age": {"$gt": 21}}   # age > 21
{"age": {"$lt": 21}}   # age < 21
{"grade": {"$ne": "C"}} # grade != C
{"grade": {"$in": ["A","B"]}} # grade is A or B
Same as Python if age > 21 but written in MongoDB's style.
